Manufacturing KPI Dashboard

OEE Analysis & Performance Tracking

Built a comprehensive KPI dashboard with 30-day simulated data to track defect rates and production line productivity.

Role Business Analyst
Timeline 4 Weeks
Tools PowerBI / Excel

Business Problem

The factory floor relied on manual data entry at the end of each shift, leading to delayed reporting, data entry errors, and inability to identify bottlenecks in real time. Defect rates were tracked in scattered Excel sheets.

As-Is (Current State)

  • Data collected on paper clipboards
  • Reports generated 24 hours late
  • No visibility into shift-by-shift OEE

To-Be (Future State)

  • Automated data ingestion from IoT sensors
  • Daily batch-update dashboard
  • Instant bottleneck and defect rate tracking

User Stories & Acceptance Criteria

Example of how requirements were handed over to the development team.

US-01: As a Production Manager, I want to view the OEE of Line A per shift, so that I can identify which shift is underperforming.
Acceptance Criteria:
  • Given the manager is on the Dashboard page
  • When they select "Line A" and "Morning Shift" from the filters
  • Then the OEE gauge chart must update to reflect the specific data for that shift within 2 seconds.

MVP Scope & Prioritization

Due to API constraints, real-time tracking was descoped. The Minimum Viable Product focused on:

1. Data Ingestion Daily batch export from legacy ERP via CSV.
2. Core Metrics OEE calculation (Availability, Performance, Quality).
3. Visualizations Pareto chart for defect reasons.
