Project 01

Personal Subscription & Expense Analytics

Centralizing fragmented financial data to identify "ghost subscriptions" and optimize monthly burn rate.

Data Analyst / BARole
2 WeeksTimeline
Excel, Power BI, FigmaTools

1. The Problem

Managing personal finances today resembles managing a poorly optimized enterprise. Money flows out through multiple channels: credit cards, e-wallets, and bank transfers. The biggest silent drain is auto-renewing subscriptions (software, streaming, gym memberships) that are rarely used but forgotten.

Without a centralized view, tracking the actual "Monthly Burn Rate" relies on memory rather than data, leading to budget overruns.

2. Constraints & Trade-offs

The Constraint: I initially wanted to build a system that auto-syncs transactions in real-time via Bank APIs. However, open banking APIs for personal use in Vietnam are highly restricted and costly.

The Trade-off: Instead of over-engineering, I opted for a Manual CSV Export MVP. I export bank statements weekly, run them through a standardized Power Query script to clean the data, and feed it into the dashboard.

Why it works: Personal finance requires behavioral reflection. A weekly manual update forces me to actually look at the numbers, satisfying the business goal better than a fully automated background process.

3. Approach & BA Artifacts

A. Data Cleaning & Standardization
Raw bank statements are messy. I built a mapping logic in Excel to categorize transactions. For example, any description containing "NETFLIX" or "SPOTIFY" is automatically tagged as Category: Entertainment and Type: Fixed Recurring.

B. Data Modeling (Star Schema)
To ensure the dashboard loads quickly and can scale if I add years of data, I structured the dataset using a Star Schema:

  • Fact Table: Transaction records (Amount, Date, ID).
  • Dimension Tables: Dim_Category (Needs vs. Wants), Dim_Date (Month, Year, Quarter).

C. Defining KPIs
Before opening the visualization tool, I defined what metrics actually drive decisions: Subscription Utilization Rate, True Fixed Costs, and Variance to Budget.

4. Core System Functions & UI Design

Beyond the interface, I defined strict functional requirements for the MVP:

  • Subscription CRUD: Full capabilities to Create, Read, Update, and Delete subscription records, tracking billing cycles natively.
  • "Ghost Detection" Algorithm: Automatically scans for services with zero transactions over 30+ days.
  • Savings Calculator: Calculates precise annual/monthly savings upon potential cancellation.
  • Exception Handling: Explicit flows for expired linked cards, failed recurring transactions, and a "Snooze" action for false positive ghost alerts.

The final product is a single-page interactive dashboard. As a UI/UX designer, I applied visual hierarchy principles and Neumorphism (Calm UX) to the data.

Real-world UI/UX Benchmark

Fintech interfaces often cause anxiety due to financial deficits. To counteract this, I benchmarked industry leaders in Calm UX and Neumorphism:

  • Cred App: Inspired the tactile, 3D soft-shadow buttons that feel like popping bubble wrap, using light/shadow to guide the user naturally without overwhelming colors.
  • Neumorphism Day and Night: Influenced the dual-layer box-shadow techniques to ensure the Neumorphic UI remains accessible and distinct across both light and dark themes.
  • Medium: Heavily influenced the minimalist approach, utilizing generous whitespace and neutral tones to reduce cognitive load to the absolute minimum.

5. BA Deliverables Deep Dive

To ensure technical feasibility, I prepared the following artifacts for the engineering team:

A. Database Schema & SQL Query (Ghost Detection)

This query drives the core logic, identifying subscriptions with no transactions in the past 30 days. It demonstrates an understanding of relational databases and handling edge cases with LEFT JOIN.

SELECT 
    s.subscription_id, 
    s.service_name, 
    s.monthly_cost
FROM 
    Subscriptions s
LEFT JOIN 
    Transactions t ON s.subscription_id = t.subscription_id 
    AND DATEDIFF(DAY, t.transaction_date, GETDATE()) <= 30
WHERE 
    t.transaction_id IS NULL 
    AND s.status = 'Active';

B. UI Wireframe Flow

I mapped out the user experience flow using Figma. When the backend flags a "Ghost", the UI triggers a non-intrusive alert modal containing:
• The specific service name and potential monthly savings.
• A clear CTA (Call to Action) guiding the user to the cancellation page.
• A "Snooze" or "Keep" option to train the algorithm on false positives.

6. Business Impact & Results

This system turned raw transaction rows into actionable financial logic.

  • Identified and canceled 4 redundant subscriptions, saving approximately 5% of monthly fixed costs.
  • Reduced time spent reviewing finances from 2 hours/month to just 15 minutes/week.

Key Takeaway: A beautiful dashboard is useless if the underlying data model is messy. Spending 80% of the project time on Power Query (data cleaning) ensured the UI was accurate and insightful.
